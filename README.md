# CookMate — AI智能厨房助手

## 一、项目简介

CookMate是一套前后端分离全栈安卓应用，以减少食物浪费为核心，接入阿里云百炼大模型实现AI菜谱生成、食材管理、社区分享等功能。采用Java安卓客户端+SpringBoot后端+MySQL云端数据库+Room本地SQLite双数据库架构，完整实现多用户数据隔离、AI多轮对话、食材保质期预警、社区互动等全套业务，适用于安卓毕业设计、课程设计。

## 二、核心功能亮点

1. AI大模型集成：依托阿里云百炼DeepSeek-V4-Pro模型，自动读取用户库存食材生成专属菜谱，支持食材储存、烹饪技巧多轮问答
2. 完整前后端分离架构：SpringBoot提供标准RESTful接口，Android端采用MVVM架构分层解耦
3. 双数据库存储方案
   云端MySQL：持久化用户、食材、菜谱、社区帖子等业务数据，不同用户数据完全隔离
   本地Room SQLite：存储AI对话历史，切换页面会话数据不丢失
4. 智能食材管理：自动计算食材保质期，临期食材置顶展示，支持食材消耗标记、数据统计、批量清空
5. 完整用户体系：账号注册、登录、自动登录、退出登录、账号注销（注销需密码二次校验）
6. 社区互动模块：用户发布图文帖子，可绑定个人菜谱，仅发帖作者拥有删除权限

## 四、全部功能模块

### 1. 用户模块

注册：校验用户名唯一性，信息写入数据库
登录：账号密码校验，返回用户ID本地持久保存
自动登录：应用启动读取本地登录标识，免重复登录
退出登录：清空本地登录信息，跳转登录页面
账号注销：输入密码二次验证，同步删除该用户所有食材、菜谱数据

### 2. 食材库存模块

新增食材：填写名称、分类、数量、购买时间，支持图片上传
自动计算保质期：根据食材类别自动生成过期时间，支持手动修改
列表排序：临期食材置顶，用不同颜色区分新鲜、临期、已消耗状态
数据统计面板：展示食材总数、即将过期数量、已消耗数量
支持长按标记消耗、单条删除、一键清空全部食材

### 3. AI智能对话模块

多轮连续对话，支持烹饪、食材储存相关咨询
自动读取当前用户库存食材，生成个性化饮食建议
侧边栏会话管理，支持新建、切换、删除历史对话
聊天气泡区分用户消息与AI回复，附带发送时间戳

### 4. AI菜谱模块

自动拉取用户全部库存食材，拼接提示词调用AI接口
解析AI返回内容，拆分菜名、食材、烹饪时长、难度、操作步骤
数据双端存储：本地Room缓存、云端MySQL永久保存
支持手动自定义录入菜谱、加载云端历史菜谱、删除菜谱

### 5. 社区模块

发布帖子：自定义标题、正文，可绑定本人已创建菜谱
全局帖子列表展示，显示发布人、发布时间、菜谱快速查看入口
权限控制：仅帖子创建者可删除自己发布内容

### 6. 页面导航结构

底部5个Tab切换，使用Fragment实现页面缓存

1. 食材库：食材管理、数据统计、新增食材
2. 菜谱：AI生成菜谱、自定义菜谱、菜谱详情
3. 社区：浏览帖子、发布帖子
4. AI：智能对话、历史会话管理
5. 我的：个人信息、退出登录、账号注销

## 五、数据库设计

### 5.1 云端MySQL 数据库名：cookmate

1. users 用户表
   存储用户ID、用户名、密码、账号注册时间
2. food_items 食材表
   关联用户ID，记录食材名称、分类、数量、购买日期、过期日期、消耗状态
3. recipes 菜谱表
   存储菜谱归属用户、菜名、食材、制作步骤、烹饪时长、难度、创建时间
4. community_posts 社区帖子表
   存储发帖用户ID、用户名、帖子标题、正文、绑定菜谱内容、发布时间

### 5.2 本地Room SQLite 数据库名：cookmate_database

chat_history 对话记录表
存储会话ID、消息发送方、消息内容、发送时间

## 六、后端接口统一规范

### 6.1 标准返回格式

{
  "code": 200,
  "message": "success",
  "data": {}
}
状态码说明
200：请求成功
400：前端参数错误
500：服务器内部异常

### 6.2 核心接口地址

### 用户相关

* 注册：`/api/user/register`
* 登录：`/api/user/login`
* 注销账号：`/api/user/{id}`

### 食材相关

* 查询用户全部食材：`/api/food/{userId}`
* 查询已消耗食材：`/api/food/consumed/{userId}`
* 新增食材：`/api/food`
* 修改食材状态：`/api/food/{id}`
* 删除单条食材：`/api/food/{id}`
* 清空用户全部食材：`/api/food/clear/{userId}`

### 菜谱相关

* 获取全部云端菜谱：`/api/recipes`
* 获取个人菜谱：`/api/recipes/user/{userId}`
* 发布菜谱：`/api/recipes`
* 删除菜谱：`/api/recipes/{id}`

### 社区帖子相关

* 获取全部帖子：`/api/posts`
* 发布帖子：`/api/posts`
* 删除帖子：`/api/posts/{id}`

## 七、项目启动步骤

### 7.1 后端服务启动

1. 本地安装MySQL8.0，新建数据库cookmate
2. 打开server文件夹，修改application.properties内数据库账号、密码
3. 终端进入server目录执行启动命令
   mvn spring-boot:run
   服务本地访问地址：localhost:5000
   安卓模拟器访问地址：10.0.2.2:5000

### 7.2 安卓客户端启动

1. Android Studio导入CookMate2项目
2. 修改网络请求地址为后端服务IP
3. 配置阿里云百炼平台API密钥，AI功能依赖该配置
4. 运行项目至MuMu模拟器或安卓真机
